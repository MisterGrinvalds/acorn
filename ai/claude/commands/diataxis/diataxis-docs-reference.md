# Diataxis: Create Reference Documentation

Create or improve **Reference** documentation for this project using Diataxis principles.

## Reference Characteristics (Information-Oriented, Theoretical)

**Purpose**: Provide technical descriptions for consultation during work

**Core Principles**:
1. **Describe only**: No instruction, explanation, or opinion
2. **Be austere and neutral**: Factual, objective tone
3. **Use standard patterns**: Consistency for predictability
4. **Mirror product structure**: Organization reflects system architecture
5. **Include examples**: Clarify without explaining
6. **Provide accuracy**: Like nutritional labelsâ€”trustworthy facts
7. **Enable consultation**: Users look up information, don't read sequentially

**Structure Requirements**:
- Technical description of machinery
- Structured like the product itself
- Standardized format across entries
- Comprehensive coverage
- Quick lookup capability

**Content Types**:
- API specifications
- Command-line flags and arguments
- Configuration options
- Function/method signatures
- Data structures
- Error codes and messages

**Language Conventions**:
- Factual statements about behavior
- Lists of features/commands/limitations
- Appropriate warnings
- No marketing or subjective claims
- Neutral, descriptive tone

**Examples for This Project**:
- "Command Reference" (all commands, flags, arguments)
- "Configuration Reference" (all config options)
- "API Reference" (if applicable)
- "Error Code Reference"
